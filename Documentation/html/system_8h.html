Enumeration Type Documentation

◆ sleep_mode_t

enum sleep_mode_t

The type used for specifying hibernate modes.

Enumerator
HIBERNATE_LIGHT 

Light sleep. All peripheral clocks remain enabled.

HIBERNATE_DEEP 

Deep sleep.

Peripheral clocks may be disabled and other power-saving features which may slow down wakeup are enabled.

HIBERNATE_MAX 

Deepest sleep.

Depending on platform and configuration wakeup may only be possible by device reset or certain specific interrupts and the contents of SRAM and device registers may be lost.

◆ uHAL_status_flag_t

enum uHAL_status_flag_t

The container type for system status flags.

Enumerator
uHAL_FLAG_SERIAL_IS_UP 

Serial communication is ready.

Attention
Set by the backend platform_init() if/when the serial communication interface is available.
uHAL_FLAG_IRQ 

A system interrupt is requested by the frontend.

Attention
Set by user code when e.g. hibernation should be interrupted. Must be cleared by user code before entering interruptable hibernation to avoid immediately exiting.
uHAL_FLAG_INHIBIT_HIBERNATION 

Disallow sleep modes that may interfere with normal functionality by (for example) disabling outputs or interrupts.

Attention
Set and cleared by user code.

◆ hibernate()

void hibernate ( sleep_mode_t  sleep_mode,
uHAL_flags_t  flags 
)

Hibernate (lower-power mode) - Untimed.

Attention
Sleeping via this function won't set up any default methods of waking, it's up to the caller to make sure that's taken care of.
On platforms that emulate an RTC, the system clock can not be updated during hibernation when initiated by this function.
Parameters
sleep_modeThe desired sleep mode. This may be overridden by either device configuration or system status flags.
flagsConfiguration flags. Currently this is unused.
